# Zanini et al. (2025) DLN Stage Coding Rubric

## Source
Zanini, L., Picano, C., & Spitoni, G. F. (2025). The Iowa Gambling Task: Men and
women perform differently. A meta-analysis. *Neuropsychology Review*, 35, 211–231.

## Unit of Analysis
**Study level**. Each study contributing an effect size (standardized mean difference
or unstandardized mean difference in IGT net scores) is coded for DLN stage based
on task variant features and sample characteristics.

## Coding Target
What is being coded: the **task features and sample characteristics** that determine
whether the IGT variant elicits dot-, linear-, or network-stage processing.

Note: We are coding whether the task *demands* a particular processing mode, not
inferring participant traits. Two studies using different IGT variants with similar
samples may receive different codes if the task features differ.

## Decision Tree

### Step 1: Does the task variant provide enriched feedback enabling integration?
Does the IGT variant include any of the following features that go beyond the
standard task and enable relational integration?
- Running totals or cumulative feedback displays
- Outcome + frequency information (not just outcome alone)
- Extended trials beyond standard 100 (150+ trials)
- Explicit description of deck contingencies (partial or full)
- Training phase with practice decks

**AND** does the sample have characteristics suggesting integrative capacity?
- Repeated/experienced IGT participants (2+ administrations)
- Mindfulness-trained or meditation-experienced samples
- Clinical populations with demonstrated insight (e.g., post-therapy)

- **Yes to task feature AND sample characteristic → Network**
- Yes to task feature only → see Step 2
- No → Step 2

### Step 2: Does the task emphasize rule-based or analytical processing?
Does the study use any of the following:
- Modified IGT with explicit frequency information (frequency-focused variants)
- Instructions emphasizing analytical strategy or "optimal" play
- Samples selected for analytical training (STEM students, finance professionals,
  mathematics majors)
- Variants that reduce ambiguity (e.g., simplified 2-deck versions, known
  probabilities)

- **Yes → Linear**
- No → Step 3

### Step 3: Standard IGT with naive participants?
Does the study use the standard 100-trial IGT (or close variant) with:
- First-time participants
- No feedback manipulation
- No explicit strategy instruction
- Standard ambiguity (participants must learn deck contingencies from experience)

- **Yes → Dot**
- If none of the above → **Mixed/Unclear** (document rationale)

## Pre-specified Coding Examples

| Study Feature | DLN Stage | Rationale |
|---|---|---|
| Standard 100-trial IGT, naive undergraduates | Dot | Reactive responding to immediate outcomes; no systematic integration |
| Standard IGT, participants explicitly told to "analyze patterns" | Linear | Rule-governed instruction creates sequential strategy |
| 200-trial IGT with running totals, experienced participants | Network | Extended feedback + experience enables relational integration |
| Modified IGT with frequency information displayed | Linear | Explicit information enables rule-based strategy without integration |
| Standard IGT, mindfulness-trained sample | Network | Mindfulness training supports affect-cognition integration |
| 2-deck simplified IGT | Linear | Reduced complexity enables rule-based optimization |
| Standard IGT, clinical sample (e.g., substance use disorder) | Mixed | Depends on specific clinical features; document |

## Theoretical Predictions by Stage

- **Dot-coded studies**: Poorest overall IGT performance (reactive responding to
  immediate gains); highest variability; sex difference reflects default
  strategy differences.
- **Linear-coded studies**: Moderate performance; high variability; performance
  collapses in ambiguity-heavy variants because rule-based strategies cannot
  capture the probabilistic deck structure.
- **Network-coded studies**: Best performance; smallest sex difference (integration
  reduces reliance on gendered default strategies); lowest variability.

## Key Prediction for Sex Differences
The sex difference (males > females) should be **largest in Linear-coded studies**
(compartmentalization creates gendered strategy divergence: males more likely to
adopt risk-seeking rules, females more likely to adopt conservative rules) and
**smallest in Network-coded studies** (integration reduces the influence of
default gendered strategies).

## Ambiguous Cases
- **Clinical populations**: Code as Mixed/Unclear unless the clinical condition
  has a clear mapping (e.g., vmPFC lesion patients → Dot, because somatic marker
  signals are absent). Document all clinical coding decisions.
- **Modified IGT variants** that don't fit cleanly: Code based on whether the
  modification increases integration demands (→ Network), increases analytical
  demands (→ Linear), or preserves reactive structure (→ Dot).

## Blinding Protocol
Coders should code study method sections (task variant, sample, instructions)
without seeing the reported effect sizes. Merge with effect sizes after coding.

## Inter-rater Reliability
- Two independent coders
- Target: Cohen's kappa >= 0.70
- Resolve disagreements by discussion; document final decision rationale
- If kappa < 0.60, recalibrate rubric before proceeding
- Maximum allowable Mixed/Unclear rate: 15% of studies

## Sensitivity Analyses
1. Reclassify all Mixed/Unclear as Dot → rerun
2. Reclassify all Mixed/Unclear as Linear → rerun
3. Drop all Mixed/Unclear → rerun
4. Report all three alongside primary analysis
