# Human Blinded Coding Protocol

## Purpose

Validate inter-rater reliability of DLN stage coding by having
independent human coders, naive to the DLN framework, apply the coding
rubrics to blinded extraction data. This addresses the limitation that
all DLN codes in the current manuscript were assigned by a single coder.

## Coder recruitment

- **Number**: 2 independent coders
- **Background**: Graduate-level training in behavioural or cognitive
  science (sufficient to comprehend task descriptions in the emotion
  regulation, interoception, and health psychology literatures).
  Domain-specific expertise is not required; coders apply a structured
  rubric with explicit decision rules rather than exercising clinical
  judgment.
- **Naivety requirement**: No prior exposure to the DLN framework or the
  companion manuscript (Wu, 2026). Coders must complete the screening
  questionnaire (`coder_screening_questionnaire.md`) confirming they
  have not read the preprint, attended any presentation on the DLN
  model, or been exposed to the framework's predictions.
- **Compensation**: Standard RA hourly rate or volunteer acknowledgement

## Materials

For each dataset, coders receive:

1. **Blinded extraction CSV**: Item identifiers and descriptions only.
   Effect sizes (d_plus, r_pooled, yi, vi), DLN stage codes, and coding
   rationales are stripped. Generated by
   `evidence_synthesis/analysis/prepare_blind_coding_materials.py`.

2. **Blinded coding rubric**: A self-contained rubric using neutral
   category labels (A / B / C) with decision questions and training
   examples. No DLN theory, predictions, or expected effect directions
   are included. Dataset-specific rubrics:
   - `blinded_coding_rubric_webb.md` (emotion-regulation strategies)
   - `blinded_coding_rubric_hoyt.md` (health outcome domains)

3. **Response sheet**: A CSV listing each item with fields for category
   assignment (A / B / C), free-text rationale, and confidence rating
   (high / medium / low):
   - `response_sheet_webb.csv`
   - `response_sheet_hoyt.csv`

## Datasets to code

| Dataset | k | Rubric file | Rationale for inclusion |
|---------|---|-------------|------------------------|
| Webb (2012) | 10 | `dln_stage_coding_manual.md` | Clearest rubric; smallest k for training |
| Hoyt (2024) | 8 | `hoyt2024_coding_rubric.md` | All r values verifiable from published Table 3 |

These two datasets were selected because they (a) have the clearest
coding rubrics, (b) span different domains (emotion regulation vs.
health psychology), and (c) together cover 18 items, providing
sufficient data for kappa estimation.

## Training procedure

1. **Rubric walkthrough** (15 minutes): The experimenter explains the
   three-question decision tree using the generic coding manual. No
   mention of specific predictions, expected patterns, or the DLN
   framework's theoretical claims.

2. **Practice items** (10 minutes): Coders independently code 2
   synthetic items not included in any dataset. Synthetic items should
   span different stages (e.g., "a startle-blink paradigm measuring
   reflexive response" and "a multi-domain quality-of-life inventory").
   Disagreements are discussed and resolved before proceeding.

3. **Calibration check**: If both coders agree on both practice items,
   proceed. If not, re-explain the decision tree and repeat with 2 new
   synthetic items.

## Coding procedure

1. Coders work independently (no communication during coding).
2. Each coder receives the blinded CSV and rubric for Dataset 1 (Webb),
   codes all items, then receives Dataset 2 (Hoyt).
3. Order is not counterbalanced (Webb first for all coders) because Webb
   is the training dataset and Hoyt is the generalization test.
4. Time limit: none (but expected ~20 minutes per dataset).
5. Coders record: (a) category assignment (A / B / C), (b) free-text
   rationale, (c) confidence rating (high / medium / low) for each item.

## Analysis

### Category mapping (applied after coding, not shared with coders)
- Category A = dot
- Category B = linear
- Category C = network

### Primary metric
- **Cohen's kappa** (Coder 1 vs. Coder 2) for each dataset
- **Cohen's kappa** (each Coder vs. author original) for each dataset

### Success threshold
- kappa >= 0.70 for both datasets (substantial agreement per Landis &
  Koch, 1977)
- If kappa < 0.70, the rubric is revised based on disagreement patterns
  and coding is repeated with new coders

### Secondary metrics
- Percent agreement (overall and per-stage)
- Confusion matrix identifying which stage pairs generate disagreements
- Comparison of coder rationales with author rationales for disagreement
  cases

### Reporting
Results will be reported in the manuscript Methods section (M10) as:
"Two independent coders, naive to the DLN framework, applied the
deposited coding rubrics to blinded extraction data. Inter-rater
agreement was kappa = X.XX (Webb) and kappa = X.XX (Hoyt). Coder-author
agreement was kappa = X.XX (Webb) and kappa = X.XX (Hoyt)."

## Timeline

| Week | Activity |
|------|----------|
| 1 | Recruit coders; generate blinded materials |
| 2 | Training session; independent coding |
| 3 | Compute kappa; analyse disagreements; revise rubrics if needed |
| 4 | Write up results; integrate into manuscript revision |

## Contingency

If kappa < 0.70 on either dataset:
1. Identify items causing disagreement
2. Revise rubric decision tree to clarify boundary cases
3. Recruit 2 new coders (not the same individuals)
4. Repeat coding with revised rubric
5. If still < 0.70, report honestly and discuss as a limitation
